# نخبة — خارطة الطريق الشاملة
**مرجع التطوير الدائم | الإصدار 1.0**

---

## التحليل الحالي

### نقاط القوة الموجودة
- بنية مكونات نظيفة ومنظمة (monorepo)
- تأثيرات scroll-reveal وpull-to-refresh
- Quick-view sheet محكم
- صفحة المنتج: معرض صور مع zoom، sticky buy bar، تنبيهات السعر
- السلة: شريط الشحن المجاني، كوبونات
- نظام تقييمات كامل
- فلاتر بحث (سعر، تقييم، خصم، الأحدث)
- عرض grid/list في البحث
- OAuth (Google/Facebook)

### الثغرات والضعف المكتشف

#### واجهة المستخدم والتجربة
| المنطقة | المشكلة |
|---|---|
| الصفحة الرئيسية | لا يوجد personalization، البانرات ثابتة، لا يوجد قسم "مقترح لك" |
| البحث | لا يوجد فلتر ماركة، لا فلتر مقاس، لا فلتر لون، البحث أساسي |
| صفحة المنتج | اختيار المقاس غير موجود في الـUI (الـDB يدعمه)، لا فيديو للمنتج، لا "اشتري مع" |
| السلة | لا "احتفظ لوقت لاحق"، لا مقارنة |
| الدفع | Apple Pay/Google Pay "قريباً"، لا تقدير موعد التسليم |
| الولاء | جدول النقاط موجود في الـDB لكن لا UI له |
| التنقل | لا breadcrumbs في صفحات التصفح |
| التصميم | لا dark mode، الـtypography يمكن رفعه، animations يمكن تعميقها |

#### الخادم والبيانات
| المشكلة | التأثير |
|---|---|
| الكوبونات مكودة يدوياً (hardcoded) | لا يمكن إدارتها |
| المخزون لا ينقص عند الطلب | بيانات غير دقيقة |
| البحث أساسي (LIKE فقط) | نتائج ضعيفة |
| لا لوحة إدارة | لا يمكن إضافة منتجات |
| الشحن ثابت (25 ر.س) | لا حساب حقيقي |
| لا استرجاعات/إلغاءات | تجربة ما بعد الشراء ضعيفة |

## خارطة الطريق — 8 مراحل

---

### المرحلة 1 — صقل التصميم البصري والـ Micro-interactions
**الهدف:** رفع جودة التصميم البصري لمستوى premium، تعميق الحركات، وتوحيد الهوية البصرية

#### 1.1 نظام الألوان والـ Typography
- [ ] إضافة `--text-display` و`--text-hero` لعناوين البانرات
- [ ] رفع وزن الخط لعناوين الأقسام من 700 إلى 800
- [ ] إضافة متغير `--gradient-premium` للعناصر الفاخرة (ذهبي غامق)
- [ ] تحسين تباين الألوان في بادجات الخصم
- [ ] إضافة `--shadow-float` للكروت العائمة

#### 1.2 تحسين الكروت (FeaturedCard / DealCard)
- [ ] إضافة تأثير hover بظل أعمق على desktop
- [ ] جعل زر "نظرة سريعة" يظهر بـ fade-in عند المرور بالماوس فقط (desktop)
- [ ] إضافة شريط تقدم المخزون `"تبقى 3 فقط"` عندما stock ≤ 5
- [ ] تحسين عرض الألوان — عرض اسم اللون نصاً بجانب الدوائر

#### 1.3 الصفحة الرئيسية — إعادة ترتيب بصري
- [ ] البانر: إضافة gradient overlay من الأسفل لنص أوضح
- [ ] FlashSaleStrip: تحسين العداد — أرقام بشكل flip-clock
- [ ] PromoBanners: إضافة أيقونات animated بدلاً من static
- [ ] SectionHeader: إضافة عداد المنتجات بجانب "عرض الكل"
- [ ] إضافة loading skeleton أكثر دقة يطابق شكل الكارد الحقيقي

#### 1.4 تأثيرات الانتقال والحركة
- [ ] تحسين RevealSection: تأثير مختلف لكل قسم (fade vs slide vs scale)
- [ ] إضافة `staggered` animation للكروت (كل كارد يظهر بتأخير 60ms عن السابق)
- [ ] تأثير ripple عند الضغط على الأزرار الرئيسية
- [ ] تحسين انتقال فتح QuickViewSheet — spring أكثر حيوية

### المرحلة 2 — البحث والاكتشاف المتقدم
**الهدف:** تحويل صفحة البحث إلى محرك اكتشاف بمستوى Shein

#### 2.1 فلاتر إضافية في البحث
- [ ] إضافة فلتر **الماركة** (checkbox متعدد من قائمة brands)
- [ ] إضافة فلتر **اللون** (دوائر ملونة قابلة للاختيار)
- [ ] إضافة فلتر **المقاس** (chips: XS / S / M / L / XL / XXL)
- [ ] إضافة فلتر **التوافر** (متوفر فقط / الكل)
- [ ] **فلتر السعر:** تحويله من نطاقات ثابتة إلى range slider تفاعلي
- [ ] Bottom sheet للفلاتر بدلاً من القائمة الجانبية الحالية

#### 2.2 نتائج البحث
- [ ] إضافة "X نتيجة" فوق الشبكة
- [ ] إضافة chip يعرض الفلاتر المفعّلة مع زر ✕ لكل فلتر
- [ ] "لا توجد نتائج" — تصميم مخصص مع اقتراحات بديلة
- [ ] تحسين list view: إضافة وصف مختصر للمنتج

#### 2.3 صفحة البحث — قسم الاكتشاف
- [ ] "الأكثر بحثاً الآن" — 6 كلمات مع أيقونة 🔥
- [ ] تحسين Trending chips — صور أكبر وعناوين أوضح
- [ ] "بحثت عنه مسبقاً" — مع زر حذف لكل عنصر

#### 2.4 API — تحسين البحث
- [ ] إضافة query params: `brand`, `color`, `size`, `minPrice`, `maxPrice`, `inStock`
- [ ] دعم بحث متعدد الكلمات (full-text search باستخدام `to_tsvector` في PostgreSQL)

### المرحلة 3 — صفحة المنتج — تجربة شراء استثنائية
**الهدف:** تحويل صفحة المنتج إلى تجربة تحويل عالية تضاهي أفضل المتاجر العالمية

#### 3.1 اختيار المقاس
- [ ] إضافة مكوّن `SizePicker` — chips أفقية (XS / S / M / L / XL)
- [ ] الأحجام غير المتوفرة: خط مائل عليها وlabel "نفد"
- [ ] زر "دليل المقاسات" يفتح bottom sheet بجدول قياسات (cm / inch)

#### 3.2 الإقناع الاجتماعي (Social Proof)
- [ ] `"X شخص يشاهد هذا المنتج الآن"` — رقم ديناميكي من الـDB (viewers counter)
- [ ] `"تم شراؤه X مرة في آخر 24 ساعة"` — من orders table
- [ ] شريط urgency: `"تبقى X قطعة فقط!"` عندما stock ≤ 5
- [ ] تحسين verified badge على التقييمات

#### 3.3 "اشترِ أيضاً" و"كثيراً ما يُشترى معه"
- [ ] قسم "منتجات مشابهة" — نفس الماركة أو نفس الفئة
- [ ] قسم "اشتراها معاً" — منتجان مع زر "أضف الاثنين للسلة"

#### 3.4 مشاركة المنتج
- [ ] زر Share يفتح native share API (واتساب، تويتر، نسخ الرابط)

#### 3.5 تحسين المعرض
- [ ] thumbnail strip أسفل الصورة الرئيسية (للتنقل السريع)
- [ ] عرض الصورة حسب اللون المختار

### المرحلة 4 — السلة والدفع — تجربة سلسة
**الهدف:** تقليل الاحتكاك في مسار الشراء وزيادة معدل الإتمام

#### 4.1 تحسين السلة
- [ ] "احتفظ لوقت لاحق" (Save for Later) — ينقل المنتج للمفضلة
- [ ] تقدير موعد التسليم: `"يصل بين X و Y"` بناءً على المدينة
- [ ] "أضف X ريال للشحن المجاني" — تحديث ديناميكي أثناء التسوق
- [ ] حذف المنتج: تأثير swipe-to-delete على الجوال
- [ ] ملخص السعر: breakdown واضح (مجموع + شحن + خصم الكوبون)

#### 4.2 تحسين الدفع (Checkout)
- [ ] خطوة 1: إضافة اختيار موعد التسليم (اليوم / غداً / خلال X أيام)
- [ ] خطوة 2: إضافة ملخص المنتجات بصور مصغرة
- [ ] تأثير success: confetti animation محسّن مع ملخص الطلب الكامل
- [ ] إضافة رقم الطلب وزر "تتبع طلبك" مباشرة بعد الإتمام

#### 4.3 الكوبونات — من hardcoded إلى DB
- [ ] إنشاء جدول `coupons` (code, discount_type, value, min_order, expires_at, usage_limit)
- [ ] API: `POST /coupons/validate` يتحقق من الجدول
- [ ] UI: عرض تفاصيل الكوبون (نوع الخصم + الحد الأدنى) عند التطبيق

### المرحلة 5 — نظام الولاء والنقاط
**الهدف:** تفعيل نظام النقاط الموجود في الـDB وبناء UI كامل

#### 5.1 واجهة النقاط
- [ ] بادج النقاط في Header (🪙 X نقطة)
- [ ] كرت النقاط في صفحة الحساب — balance + تاريخ الكسب
- [ ] عرض "ستكسب X نقطة" على كل منتج في صفحة التفصيل
- [ ] عرض "كسبت X نقطة!" في شاشة نجاح الطلب

#### 5.2 استبدال النقاط
- [ ] خطوة الدفع: checkbox "استخدم نقاطي (X نقطة = X ريال)"
- [ ] API: تعديل `POST /orders` ليدعم خصم النقاط

#### 5.3 إنجازات الولاء (Tiers)
- [ ] تعريف 3 مستويات: عضو / ذهبي / بلاتيني (بناءً على إجمالي المشتريات)
- [ ] بادج المستوى في صفحة الحساب
- [ ] شريط تقدم نحو المستوى التالي

### المرحلة 6 — التخصيص والتوصيات
**الهدف:** "الصفحة الرئيسية التي تعرفك" — محتوى مخصص لكل مستخدم

#### 6.1 قسم "مقترح لك" في الصفحة الرئيسية
- [ ] خوارزمية بسيطة: منتجات من نفس فئات المنتجات المشاهدة مؤخراً
- [ ] يظهر فقط إذا كان للمستخدم history (مشاهدة أو شراء)
- [ ] شريط أفقي بـ FeaturedCard

#### 6.2 "أكمل ما بدأته"
- [ ] كرت في الصفحة الرئيسية يعرض آخر منتج في السلة مع زر "أكمل الطلب"

#### 6.3 تحسين "شاهدته مؤخراً"
- [ ] الحد الأقصى 10 منتجات
- [ ] زر "مسح السجل"
- [ ] يظهر فقط إذا كان هناك ≥ 2 منتجات

### المرحلة 7 — تتبع الطلبات وما بعد الشراء
**الهدف:** تجربة ما بعد الشراء بمستوى Amazon/Noon

#### 7.1 تتبع الطلب البصري
- [ ] Timeline مرحلي: تم الطلب ← قيد التحضير ← في الطريق ← تم التسليم
- [ ] كل مرحلة بأيقونة ملونة وتاريخ
- [ ] progress bar أفقي يربط المراحل

#### 7.2 إدارة الطلبات
- [ ] زر "إعادة الطلب" في الطلبات المكتملة — يضيف نفس المنتجات للسلة
- [ ] زر "إلغاء الطلب" للطلبات الجديدة فقط (status = pending)
- [ ] API: `PATCH /orders/:id/cancel`

#### 7.3 طلب الاسترجاع
- [ ] نموذج استرجاع بسيط: السبب + صورة اختيارية
- [ ] DB: جدول `returns` (order_id, reason, status, created_at)
- [ ] API: `POST /orders/:id/return`

### المرحلة 8 — الميزات المتقدمة (Premium Layer)
**الهدف:** ميزات تجعل التطبيق يتفوق على المنافسين

#### 8.1 مقارنة المنتجات
- [ ] زر "قارن" على كل كارد (يُضاف للـ context)
- [ ] Bottom sheet مقارنة: جانب لجانب (حتى 3 منتجات)
- [ ] جدول: السعر، التقييم، الماركة، الخصم، الألوان

#### 8.2 حصص المنتجات (Flash Deals Slots)
- [ ] قسم "عرض الساعة" — صفقة واحدة تتغير كل ساعة
- [ ] عداد دقيق للثواني مع تأثير flip
- [ ] الكمية المتبقية بشريط تقدم

#### 8.3 Dark Mode
- [ ] تعريف مجموعة CSS variables بديلة تحت `[data-theme="dark"]`
- [ ] زر toggle في صفحة الإعدادات
- [ ] حفظ التفضيل في localStorage

#### 8.4 Referral Program
- [ ] كود الإحالة في صفحة الحساب (قابل للنسخ والمشاركة)
- [ ] DB: حقل `referred_by` في users + جدول `referrals`
- [ ] مكافأة: 50 نقطة لكل إحالة ناجحة
