# MINIMAX - Professional Monorepo Architecture

هذا المشروع يمثل النسخة النهائية، النظيفة، والمهيكلة بأعلى معايير الاحترافية البرمجية. تم إجراء تنظيف جذري وعميق (Deep Scrub) لكافة الملفات غير الضرورية والمكررة.

## الهيكل المعماري (Architecture)

المشروع مبني بنظام **Monorepo** متطور يضمن فصل الاهتمامات (Separation of Concerns):

### 📱 التطبيقات (Apps)
- **`apps/vibe-app`**: التطبيق الرئيسي للواجهة الأمامية (Frontend).
- **`apps/api-server`**: خادم الواجهة البرمجية (Backend API).
- **`apps/mockup-sandbox`**: بيئة تطوير واختبار المكونات الرسومية.

### 📦 الحزم والمكتبات (Packages)
- **`packages/db`**: طبقة الوصول للبيانات باستخدام Drizzle ORM.
- **`packages/api-spec`**: تعريفات OpenAPI الموحدة.
- **`packages/api-client-react`**: عميل API مولد تلقائياً لـ React.
- **`packages/api-zod`**: مخططات التحقق من البيانات باستخدام Zod.

### 🛠️ الأدوات (Tools)
- **`scripts/`**: سكربتات الصيانة والأتمتة.

## التحسينات الجذرية:
1. **تنظيف جذري (Deep Scrub):** حذف كافة ملفات الكاش، السجلات، والملفات المخفية غير الضرورية.
2. **بنية معيارية:** تنظيم الملفات وفقاً لأفضل ممارسات المشاريع الكبرى.
3. **تكوين مثالي:** تحديث كافة ملفات `tsconfig` و `package.json` لضمان توافق المسارات.
4. **جاهزية الإنتاج:** المشروع الآن في حالة مثالية للبدء في التطوير أو النشر.
